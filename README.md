# みんなのタブ

「AIを使った開発」講座のワーク用リポジトリです。参加者それぞれが1つのタブ（1機能）を実装し、最終的に全員の機能が1つのWebサイトにまとまります。

デモ: `index.html` をブラウザで開く、または GitHub Pages の公開URL（講師が案内します）

---

## 構成

```
work1-site/
├── README.md      # このファイル（当日の手順書）
├── index.html     # ホーム画面。tabs.js を読んでタブボタンを並べる
├── style.css      # 共通スタイル（触らない）
├── tabs.js        # ★タブ登録リスト。1人1行だけ追加する
└── tabs/
    ├── _sample.html   # 見本タブ（コピーして自分のタブを作る）
    └── (ここに あなたの名前.html を追加する)
```

**あなたが触るのは `tabs/自分の名前.html`（新規作成）と `tabs.js`（1行追加）だけです。**

---

## ワークのルール

1. **mainに直接pushしない**（設定でできないようにしてあります）
2. **自分のファイルだけ触る**: `tabs/自分の名前.html` の新規作成と `tabs.js` への1行追加のみ。`git add` は個別ファイル指定で行う
3. **説明できないコードはコミットしない**: PRの説明欄に「AIをどこに使ったか」を1行書く

**安全ルール**: APIキー・パスワードをAIに貼らない／AIの出したコマンドは内容を確認してから実行する

## 手順

### Step 1: ブランチを作る

```bash
git clone <このリポジトリのURL>
cd work1-site
git switch -c feat/自分の名前
```

`index.html` をブラウザで開く（またはLive Server）。見本タブが動くことを確認してください。

### Step 2: 自分のタブを作る

1. `tabs/_sample.html` をコピーして `tabs/自分の名前.html` を作る
2. 作りたい機能を**日本語3行の仕様**にしてからAIに依頼する
   - 例: 「おみくじ。ボタンを押すと大吉〜凶をランダム表示。連打できる。HTML1ファイル完結、外部ライブラリなし」
3. AIへの依頼には**制約を必ず伝える**:
   - 「HTML1ファイルで完結」
   - 「外部CDN・ライブラリ禁止」
   - 「このファイルはiframeで表示される」
4. ブラウザで動作確認
5. `tabs.js` に自分の1行を追加する

```js
const TABS = [
  { name: "見本: あいさつ", file: "tabs/_sample.html", author: "先生" },
  { name: "おみくじ", file: "tabs/yamada.html", author: "山田" },  // ← こう追加する
];
```

6. `index.html` を開き直し、ホーム画面から自分のタブが開けることを確認

> 時間内に完成しなければ機能を削ってください。**「動く小さいもの」＞「動かない大きいもの」**

### Step 3: push して PR を出す

```bash
git add tabs/自分の名前.html tabs.js      # 個別指定。git add -A は禁止
git commit -m "feat: ◯◯タブを追加"
git push -u origin feat/自分の名前
```

GitHubでPull Requestを作成し、説明欄に以下を記入してください。

```
## 何を作ったか

## なぜこの機能か（1行）

## AIをどこに使ったか（1行）

## 自分で確認したこと（例: ボタン連打しても壊れない）
```

### Step 4: 隣の人のPRをレビューする

チェックリスト:

- [ ] ブランチを手元に持ってきて実際に動かした（`git fetch` → `git switch feat/相手の名前`）
- [ ] 変更ファイルが「本人のタブ＋tabs.jsの1行」だけになっている
- [ ] 分からない行を1つ質問した（作者が自分の言葉で答えられたらOK）

問題なければ Approve。承認が付いたら講師の合図でマージします。

### Step 5: 統合の瞬間

全員のPRがマージされたら:

```bash
git switch main
git pull
```

手元のブラウザに、**全員のタブが載ったサイト**が現れます。GitHub Pagesの公開URLでも同じものが見られます。
